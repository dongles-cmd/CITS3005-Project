Project Overview

This project is part of the CITS3005 course and is worth 30% of the final grade, due October 18, 2024. It focuses on building a knowledge graph and ontology for the iFixit dataset, representing repair procedures for various devices.

The project involves:

  • Creating an ontology to describe concepts like procedures, items, tools, and steps.
  • Populating the ontology with data from the iFixit dataset and running meaningful queries.
  • Building a lightweight application (command-line or Flask-based) to browse and search the knowledge graph.
  • Writing a user manual and report outlining the design and execution process.

Features

  • Ontology: Describes procedures, tools, items, and steps, enforcing rules like tool usage and part-subprocedure relationships.
  • Knowledge Graph: Populated with iFixit data, searchable via SPARQL queries.
  • Search: Find procedures, items, tools, and steps using keyword or structured search.
  • Error Checking: Automatically identify inconsistencies in the data based on the ontology.

Project Resources

  • Owlready2 for ontology management and reasoning
  • RDFLib for knowledge graph management and SPARQL queries
  • Flask for building the web interface
  • iFixit Data for the knowledge base
